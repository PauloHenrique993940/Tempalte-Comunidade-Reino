Comunidade do Reino

Documento da Área Administrativa

Versão de funcionamento atual, escopo do MVP e próximos passos recomendados para evolução do sistema administrativo.

1. O que o projeto faz no momento

O projeto, no estado atual, funciona como um portal institucional estático da Comunidade do Reino. Ele apresenta páginas públicas para divulgar a comunidade, projetos, biblioteca, podcasts, voluntariado, contato, políticas e páginas de acesso.

Além do portal público, foi criada uma primeira versão visual da área administrativa. Essa versão é um MVP demonstrativo, ou seja, mostra a estrutura, os módulos e os fluxos esperados, mas ainda não possui banco de dados, autenticação real ou persistência de informações.

Portal públicoPermite navegar por páginas institucionais, conhecer projetos, biblioteca, podcasts, voluntariado e canais de contato.
Painel administrativo MVPApresenta uma tela administrativa inicial para validar dashboard, cadastros, módulos e organização visual.
Formulários demonstrativosOs formulários exibem mensagens de confirmação, mas ainda não salvam dados em backend.
Navegação de acessoO cabeçalho possui uma seleção única para escolher entre acesso de Voluntário e Admin.

2. Versão de funcionamento atual

Área Status atual Observação
Site institucional Funcional como site estático As páginas HTML podem ser abertas e navegadas localmente ou publicadas em hospedagem estática.
Área administrativa MVP visual/demonstrativo Existe a página admin.html com módulos principais, dados simulados e interface responsiva.
Login administrativo Demonstração O formulário existe, mas não valida usuário, senha, sessão ou permissão real.
Banco de dados Não implementado Ainda não há tabelas, API ou armazenamento persistente para usuários, voluntários e eventos.
Segurança e LGPD Planejada A interface já prevê logs e perfis, mas as regras reais precisam ser implementadas no backend.

3. Objetivo da Área Administrativa

Centralizar a administração do site e fornecer ferramentas que permitam aos líderes e administradores gerenciar as atividades da Comunidade com eficiência, segurança e organização.

4. Funcionalidades previstas

4.1 Dashboard

  • Indicadores gerais do sistema.
  • Número de usuários cadastrados.
  • Voluntários ativos.
  • Novos cadastros.
  • Estatísticas de acesso.
  • Avisos e notificações.

4.2 Gestão de Usuários

  • Cadastro de usuários.
  • Edição e atualização de dados.
  • Ativação e desativação de contas.
  • Recuperação de acesso.
  • Histórico de atividades.

4.3 Perfis de Acesso

  • Administrador Geral.

  • Líder de Departamento.
  • Voluntário.
  • Usuário da Comunidade.

4.4 Gestão de Departamentos

4.5 Cadastro de Voluntários

4.6 Biblioteca Digital

  • Upload de documentos.
  • Organização por categorias.

4.7 Conteúdo do Site

  • Notícias.
  • Artigos.

  • Publicações institucionais.

4.8 Formulários e Questionários

  • Criação de formulários.

  • Visualização das respostas.
  • Exportação dos dados.

4.9 Agenda

  • Eventos.
  • Reuniões.

  • Calendário institucional.

4.10 Relatórios

  • Exportação de dados.
  • Estatísticas.
  • Indicadores de participação.
  • Relatórios administrativos.

4.11 Configurações Gerais

  • Dados institucionais.
  • Redes sociais.
  • Contatos.

4.12 Segurança

5. O que já foi entregue na primeira etapa

  • Criação da página administrativa admin.html.
  • Dashboard demonstrativo com indicadores iniciais.
  • Estrutura de menu administrativo com módulos do MVP.
  • Listagem demonstrativa de usuários.
  • Formulários demonstrativos para usuários, voluntários e configurações.
  • Seções para departamentos, conteúdo, agenda, segurança, logs e LGPD.
  • Seleção única no cabeçalho para acesso de Voluntário ou Admin.
Importante: esta etapa não compromete decisões definitivas de backend, banco de dados ou arquitetura. Ela funciona como protótipo funcional de navegação e validação visual para orientar a próxima fase.

6. O que ainda falta para virar sistema real

  1. Implementar autenticação real com senha criptografada.
  2. Criar backend/API para salvar e consultar dados.
  3. Definir e criar o banco de dados.
  4. Implementar permissões por perfil de acesso.
  5. Conectar formulários administrativos ao backend.
  6. Criar logs reais de ações administrativas.
  7. Aplicar políticas de segurança, backup e LGPD.
  8. Testar com usuários reais antes da publicação oficial.

7. Estratégia de Desenvolvimento

A recomendação é seguir uma estratégia incremental, iniciando com um MVP contendo apenas os recursos essenciais para colocar o sistema em funcionamento. A partir dessa base, novas funcionalidades poderão ser adicionadas conforme a Comunidade crescer e novas demandas surgirem.

Para a próxima fase, a prioridade recomendada é implementar login administrativo real, usuários, perfis de acesso e departamentos. Essa base sustenta os demais módulos com mais segurança.

8. Visão de Longo Prazo

O sistema deve ser pensado para acompanhar o crescimento da Comunidade do Reino nos próximos anos. Uma arquitetura modular permitirá incorporar novos recursos sem reconstruir toda a plataforma.

  • Área de cursos e treinamentos.
  • Biblioteca digital ampliada.
  • Gestão de podcasts.
  • Emissão automática de certificados.
  • Gestão avançada de eventos.
  • Área exclusiva para líderes e voluntários.
  • Integrações futuras com outros serviços.
  • Aplicativo móvel.

9. Como gerar o PDF manualmente

  1. Abra este arquivo no navegador: DOCUMENTO-AREA-ADMINISTRATIVA.html.
  2. Pressione Ctrl + P.
  3. Em destino, escolha Salvar como PDF.
  4. Confirme o tamanho A4 e salve o arquivo.
